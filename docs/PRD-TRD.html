Executive Summary
Problem, Vision, and Success Metrics

A local election candidate contesting Constituency S20-155 (Part 275) faces an operational challenge: managing a field army of 300+ campaign workers covering 300+ wards with ~70,000 registered voters — all currently tracked on printed paper lists. This is error-prone, unscalable, and provides zero real-time visibility.

The Election Campaign Field Operations Manager is a digital platform that replaces this paper workflow entirely — enabling real-time voter tracking, hierarchical team management, and live campaign analytics accessible from any mobile device or desktop.

ℹ️
Voting Mechanism Context: Eligible voters (age 18–35) must record a video on a government application stating their vote. Field workers visit each voter, explain the process, and record their status. This unique workflow is central to the system design.

Stakeholders & User Personas
Who uses this system and what they need
Super Admin (Campaign Leader)
Profile: Senior campaign coordinators, up to 304 individuals

Goals:
  • Full visibility into campaign progress across all wards
  • Assign and manage field teams efficiently
  • Identify problem areas and top performers instantly
  • Communicate with field workers at scale

Pain Points (current): No real-time data, manual phone calls for updates, no way to prevent duplicate work
Field Worker (Ground Team)
Profile: Door-to-door campaigners, primarily mobile-only users

Goals:
  • Know exactly which voters to visit today
  • Mark voter status quickly on the go
  • Avoid re-visiting already-covered voters
  • Receive instructions from campaign leadership

Pain Points (current): Paper lists get lost, no way to coordinate with co-workers, no credit tracking
Sub-Worker (Junior Field Member)
Works under a Field Worker. Same interface as Field Worker but their vote credits roll up to their parent worker's totals. Created by Field Workers or Super Admins.

User Stories
Key workflows described as user stories with acceptance criteria

Super Admin Stories

US-A01: As a Super Admin, I want to upload an Excel file of voter data so that I can digitize the paper voter roll without manual entry.
AC: System accepts .xlsx/.xls/.csv; auto-detects Name/Age/VoterID columns; shows preview; imports with duplicate skipping; displays count of eligible (18–35) voters.
US-A02: As a Super Admin, I want to assign groups of voters to specific workers so that each worker has a defined territory to cover.
AC: Can select multiple voters via checkbox; assign all at once to any worker; can also auto-assign all unassigned voters in an area to a chosen worker.
US-A03: As a Super Admin, I want to see a real-time dashboard so that I can monitor campaign progress without calling individual workers.
AC: Dashboard shows total done/refused/pending across all areas; area-wise bar chart; worker leaderboard; today's activity feed; updates without full page reload.
US-A04: As a Super Admin, I want to send broadcast notifications so that I can communicate instructions to all field workers instantly.
AC: Can send to all workers or filter by specific area; notification appears in worker's inbox; read/unread tracking per worker.

Field Worker Stories

US-W01: As a Field Worker, I want to see only my assigned voter list when I log in so that I know exactly who to visit without any confusion.
AC: Shows personal stats (total/done/refused/pending); displays voter cards with address; filter by status; no other worker's data visible.
US-W02: As a Field Worker, I want to mark a voter as Done with one tap so that I can update the record quickly while standing at the voter's door.
AC: Green check button marks as Done; voter card turns green; status visible to all co-workers on shared list immediately; credited to my name.
US-W03: As a Field Worker sharing a list with 2 other workers, I want to see when another worker marks a voter so that I don't waste time visiting already-covered voters.
AC: Within 30 seconds of another worker marking a voter, the change is reflected on my list without page refresh. Marked voter shows "by [Worker Name]".
US-W04: As a Field Worker, I want to show a how-to video to the voter so they know how to record their vote correctly.
AC: Video Guide section in menu; videos play in full-screen modal; works on mobile without download.

Key Design Decisions

DecisionChoiceRationale
DatabaseSQLite (sql.js)Zero config, single file, no native compilation. Upgrade path to PostgreSQL exists.
Auth strategyJWT (stateless)No server-side session storage needed. Token in localStorage, injected via Axios interceptor.
Dev/Prod splitVite Proxy (dev) / Serve static (prod)Backend serves built React app in prod (single URL/port). Vite proxies API in dev for HMR.
CSS frameworkTailwindCSSUtility-first, smaller bundle, more customizable than Bootstrap, mobile-first by design.
File uploadsMulter + disk storageVideos stored on filesystem; metadata in SQLite. Simple, no cloud dependency for v1.
Real-time syncPolling (30s)Simpler than WebSockets. Adequate for campaign use case. Can upgrade to SSE/WebSocket in v2.

Security Architecture
Authentication, authorization, data protection, and audit

Authentication

  • ✓
    JWT Tokens (7-day expiry)Signed with server secret; required on all /api/* except /login
  • ✓
    bcrypt Password HashingSalt rounds = 10. Passwords never stored or logged in plaintext
  • ✓
    Auto Token Expiry HandlingAxios 401 interceptor redirects to login automatically

Authorization (RBAC)

  • ✓
    Role MiddlewarerequireAdmin() middleware blocks field workers from admin routes
  • ✓
    Data IsolationWorkers can only see voters assigned to them or their sub-workers
  • ✓
    Protected Routes (Frontend)ProtectedRoute.jsx prevents direct URL access to unauthorized pages

Data Security

  • ✓
    No Voter Data in URLsAll sensitive data in request body (POST) or Authorization header; never in query string
  • ✓
    Parameterized SQL QueriesAll SQLite queries use parameters — prevents SQL injection
  • ✓
    Server-Side ValidationAll inputs validated on the server regardless of frontend checks
  • ✓
    Activity Audit LogEvery voter status change, upload, and login logged with user ID + timestamp
  • !
    CORS Restriction (v2)Currently permissive for dev. Should be locked to production domain before go-live
  • !
    HTTPS Enforcement (v2)Requires SSL certificate in production. Use Nginx reverse proxy or localtunnel for testing
